#### 트러블슈팅

**열화상 노이즈 문제 (2025.06.18)**

> 증상: 열화상 이미지 처리 결과에 노이즈가 발생  
> 원인 분석: int형으로 픽셀값을 계산하면서 소수점이 잘려 오차 누적  
> 해결: float형으로 전환하여 `((temp - min) / (max - min)) * 255.0` 연산 수행

**Nginx 설치 권한 문제 (2025.06.13~17)**

> 증상: Linux 서버에 Nginx 설치 권한 없어 `nginx` 명령어 사용 불가  
> 원인 분석: PATH 환경변수에 Nginx 실행 파일 경로 미등록, sudo 권한 제한  
> 해결: root 권한 문의 후 획득 → EPEL 저장소 설치 → Nginx 수동 설치 → SSL 인증서 발급(`/opt/apache/conf/` 경로 활용) → curl 테스트 통과

**팔레트 Enum 파싱 오류로 인한 DB DataError (2026.05.20)**

> 증상: 팔레트 열거형 파싱 시 `-1` 오프셋이 DB 삽입 단계에서 DataError 발생  
> 원인 분석: 레거시 버전 대응 코드에서 팔레트 인덱스에 -1을 빼는 보정 로직이 v2.0.0 이상에도 적용되고 있었음  
> 해결: 버전 분기 처리로 v2.0.0 미만에만 -1 오프셋 보정 적용, 이상 버전은 기준선 인덱스 유지

**비동기 파이프라인 조기 상태 업데이트 문제 (2026.05.21)**

> 증상: 업로드 파이프라인 처리 중간에 DB 상태가 완료로 업데이트되어 실제 완료 여부와 불일치 발생  
> 원인 분석: 검증기(`validator.py`)가 파이프라인 중간에 DB 상태를 갱신하는 구조  
> 해결: 초기 상태를 `is_corrupted=2 / IN_PROGRESS`로 고정하고, 파이프라인 완료 시점에만 일괄 업데이트하는 `_finalize_upload_history_status` 함수 도입. 예외 발생 시 Failed 상태로 전환 보장

**422 유효성 검사 오류 (2026.05.26)**

> 증상: 비디오 프레임 FFT 요청/응답 시 422 Unprocessable Entity 반복 발생  
> 원인 분석:  
> - `FrameFftRequest.time` 필드가 `str | int | float` 혼용 케이스를 처리하지 못함  
> - `fft_array` 필드가 `List[float]` 단순 타입이어서 백엔드 직렬화 시 `ResponseValidationError` 발생  
> 해결:  
> - `time` 필드에 `str | int | float` 유니온 타입 허용 + 공백·빈값 정규화 Pydantic 검증기 추가  
> - `fft_array`를 `List[FrameFftPoint]` (`x`, `y` 좌표) 구조체로 대체

**동영상 파일 매칭 실패 (2026.05.28)**

> 증상: `.mp4` 확장자 파일 중 `_V` 접미사 없는 케이스에서 "missing file" 오판단 발생  
> 원인 분석: 파일 매칭 로직이 `*_V.mp4` 패턴만 인식하도록 구현되어 있었음  
> 해결: 모든 `.mp4`·`.avi` 파일을 대소문자 구분 없이 수용하도록 매칭 패턴 완화. FFmpeg 동일 소스 덮어쓰기 방지 예외 방어 로직 추가
